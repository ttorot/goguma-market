# 고구마마켓 개발 규칙

## 프로젝트 개요
- **목적**: 개발 연습용 프로젝트
- **기술 스택**: Next.js (화면 구성), Supabase (데이터베이스·로그인), Vercel (인터넷 배포)

## 소통 방식

- 개발 용어는 비개발자도 이해할 수 있게 쉬운 말로 설명한다.
  - 예: "컴포넌트" → "화면 조각", "API" → "데이터를 주고받는 통로", "배포" → "인터넷에 올리기"
- 지금 무엇을 하고 있는지 자주 알려준다. (단계마다 한 줄로 현재 작업 안내)
- 작업이 끝나면 결과를 간단하게 요약해준다.

## 작업 방식

- 사용자가 지시한 것만, 지시한 순서대로 단계적으로 수행한다. 범위를 넘는 추가 작업은 하지 않는다.
- 자동으로 할 수 있는 것은 최대한 직접 처리한다.
- 사람이 직접 해야 하는 것(Supabase 대시보드 설정, Vercel 환경변수 입력, 비밀번호 입력 등)은 빠뜨리지 않고 알기 쉽게 단계별로 안내한다.

## 개발 완료 후

- 작업이 마무리되면 항상 브라우저나 미리보기를 열어서 바로 확인할 수 있게 해준다.
- 로컬 개발 서버(`npm run dev`)를 실행하고 결과 화면을 보여준다.
